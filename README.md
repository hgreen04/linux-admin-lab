# Linux Administration Lab

Production-style Linux system administration and AWS infrastructure practice.

---

## Overview

This repository documents hands-on Linux administration exercises designed to simulate real-world system administration tasks.

Environment:
- RHEL / Ubuntu virtual machines
- AWS EC2 instances
- Local lab via VirtualBox
- SSH remote management

---

## Lab Objectives

- User and group management
- File permissions and ACL configuration
- Package installation and repository management
- Service management (systemctl)
- Log analysis and troubleshooting
- Network configuration and diagnostics
- SSH hardening
- Basic firewall configuration
- AWS EC2 deployment and secure access

---

## Completed Labs

### Lab 1 – User & Permission Management
- Created users and groups
- Configured sudo access
- Set directory permissions using chmod
- Applied ACLs using setfacl
- Verified access control with getfacl

### Lab 2 – Service Management
- Installed and enabled nginx
- Configured systemd service behavior
- Analyzed logs using journalctl
- Tested service failure recovery

### Lab 3 – AWS EC2 Deployment
- Launched EC2 instance
- Configured SSH key authentication
- Hardened SSH settings
- Configured basic firewall rules
